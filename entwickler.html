🤝 Mitwirken / Contribution Guide

Wir freuen uns über jede Unterstützung! Hier erfährst du, wie du dich einbringen kannst.

🔱 Fork erstellen

Du kannst das Repository auf GitHub forken und lokal bearbeiten. So behältst du deine Änderungen separat und kannst sie später per Pull Request einreichen.

  • Gehe zu SchulSystem Repository
  • Klicke auf Fork (oben rechts)
  • Arbeite in deinem Fork wie gewohnt weiter

🌿 Branching-Konzept

Bitte verwende für neue Features oder Bugfixes immer einen separaten Branch mit sinnvollem Namen, z. B.:

  • feature/darkmode
  • bugfix/login-failure

Arbeite niemals direkt auf main oder master!

📥 Pull Requests

Wenn du mit einem Feature oder Bugfix fertig bist, erstelle bitte einen Pull Request (PR):

  • Beschreibe, was du geändert hast
  • Verlinke ggf. das zugehörige Issue
  • Nutze sprechende Titel
  • Teste deinen Code vor dem Einreichen

🐛 Issue-Tracker

Fehler oder Featurewünsche können direkt auf GitHub gemeldet werden: ➡️ Zum Issue-Tracker

  • Nutze Labels wie bug, enhancement, question
  • Beschreibe dein Problem möglichst genau

✨ Featurewünsche

Du hast eine gute Idee für eine neue Funktion? Dann eröffne ein Issue mit dem Label enhancement und beschreibe deine Idee.

🧹 Bugs melden

Wenn etwas nicht funktioniert, beschreibe bitte:

  • Was genau passiert (inkl. Fehlermeldung oder Screenshot)
  • Wie man den Fehler reproduzieren kann
  • Deine Systeminformationen (Windows/Linux, SchulSystem-Version etc.)
